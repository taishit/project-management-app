# 01 要件定義

## 目的

ProjectFlow は、複数プロジェクトの進捗確認と課題管理をシンプルに行うための Web アプリです。

## 想定ユーザー

- 開発メンバー
- プロジェクト管理者
- 閲覧中心の関係者

## 主要機能

- ログイン
- ダッシュボード表示
- プロジェクト一覧表示
- 課題一覧表示
- 課題詳細表示

## 非機能要件

- 公開 GitHub リポジトリで管理できる構成
- モノレポで frontend と backend を分離
- ローカルの PostgreSQL と連携しやすい構成
- REST API ベースで画面連携しやすい構成

## スコープ外

- 本番認証基盤との連携
- 通知配信
- ファイル添付
- 高度な権限制御
