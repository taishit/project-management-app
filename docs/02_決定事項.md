# 02 決定事項

## アプリケーション方針

- アプリケーション表示名は `ProjectFlow` とする。
- リポジトリ名は `project-management-app` とする。
- 社内情報、顧客情報、個人情報は含めない。
- Backlog や Jira の完全コピーではなく、ProjectFlow 独自の表現にする。

## 実装方針

- 今回のMVPでは認証・認可を実装しない。
- REST API のパスは `/api` 配下に統一する。
- Spring Data JPA は使用せず、MyBatis を使用する。
- SQLはMapper interfaceへ直接書かず、`resources/mapper/*Mapper.xml` に記述する。
- DB初期化は `resources/sql/schema.sql` と `resources/sql/data.sql` で行う。
- Flyway / Liquibase は今回導入しない。

## 追加開発候補

- コメント機能
- 添付ファイル機能
- 課題検索、タグ、カンバンボード、ガントチャート
- 通知、Wiki、課題履歴
- ログイン、ユーザー管理、権限管理
- CSV出力、期限切れ課題の強調表示
