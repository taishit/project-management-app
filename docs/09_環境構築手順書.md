# 09 環境構築手順書

作成日: 2026-07-04

## 1. 対象プロジェクト

この手順書は、`project-management-app` をローカル環境で開発・起動するための環境構築手順をまとめたものです。

```text
project-management-app/
├─ backend/   Java 21 + Spring Boot + Maven
├─ frontend/  Vue 3 + Vite + TypeScript
└─ docs/      設計書・手順書
```

## 2. 必要なインストール資源

未インストールの場合は、以下をインストールしてください。

| 資源 | 用途 | 備考 |
| --- | --- | --- |
| Visual Studio Code | 開発エディタ | Java / Vue / Spring Boot 開発で使用 |
| Git for Windows | ソース管理 | `git clone`、差分確認、コミットで使用 |
| JDK 21 | Backend 実行・ビルド | Oracle JDK 21 をインストールし、`JAVA_HOME` を JDK 21 に設定 |
| Apache Maven 3.9.16 | Backend 依存関係解決・起動 | このリポジトリには Maven Wrapper は含まれていないため、Maven 本体が必要 |
| Node.js 22 系 | Frontend 実行・ビルド | npm 同梱版を使用 |
| PostgreSQL 16.14 | ローカル DB | EnterpriseDB のダウンロードページから 16.14 を選択 |
| pgAdmin または psql | DB 操作 | DB・ユーザー作成、接続確認で使用 |

## 3. VS Code 拡張機能

開発に必要な VS Code 拡張機能は以下です。

| 拡張機能 ID | 用途 |
| --- | --- |
| `ms-ceintl.vscode-language-pack-ja` | VS Code 日本語化 |
| `openai.chatgpt` | ChatGPT 連携 |
| `vscjava.vscode-java-pack` | Java 開発パック |
| `vmware.vscode-boot-dev-pack` | Spring Boot 開発パック |
| `vue.volar` | Vue / TypeScript サポート |

別端末で同じ拡張機能を入れる場合は、コマンドプロンプトで以下を実行します。

```cmd
code --install-extension ms-ceintl.vscode-language-pack-ja
code --install-extension openai.chatgpt
code --install-extension vscjava.vscode-java-pack
code --install-extension vmware.vscode-boot-dev-pack
code --install-extension vue.volar
```

## 4. VS Code ワークスペース設定

現在の `.vscode/settings.json` には以下が設定されています。

```json
{
  "java.compile.nullAnalysis.mode": "automatic",
  "java.configuration.updateBuildConfiguration": "interactive"
}
```

Java プロジェクトの依存関係を変更した場合、VS Code 上で Java のビルド構成更新を求められたら許可してください。

## 5. Backend 環境構築

### 5.1 Java 21 のインストール

JDK 21 が未インストールの場合は、以下の Oracle JDK 21 インストーラーをダウンロードしてインストールします。

```text
https://download.oracle.com/java/21/latest/jdk-21_windows-x64_bin.exe
```

インストール後、`JAVA_HOME` を JDK 21 のインストール先に設定し、`Path` に `%JAVA_HOME%\bin` を追加します。

例:

```text
JAVA_HOME=C:\Program Files\Java\jdk-21
Path=%JAVA_HOME%\bin
```

既に Java 21 が入っている場合も、`JAVA_HOME` が Java 21 を指していることを確認してください。

### 5.2 Maven 3.9.16 のインストール

Maven が未インストールの場合は、以下の Apache Maven 3.9.16 の zip をダウンロードして展開します。

```text
https://dlcdn.apache.org/maven/maven-3/3.9.16/source/apache-maven-3.9.16-src.zip
```

展開後、Maven の配置先を `MAVEN_HOME` に設定し、`Path` に `%MAVEN_HOME%\bin` を追加します。

例:

```text
MAVEN_HOME=C:\apache-maven-3.9.16
Path=%MAVEN_HOME%\bin
```

既に Maven が入っている場合も、`mvn -version` で 3.9 系が利用されていることを確認してください。

### 5.3 Java / Maven の確認

コマンドプロンプトで以下を実行します。

```cmd
java -version
mvn -version
```

Java は 21 系、Maven は 3.9 系が表示されれば問題ありません。

### 5.4 PostgreSQL 16.14 のインストール

PostgreSQL が未インストールの場合は、以下の EnterpriseDB のダウンロードページを開き、Windows 用の PostgreSQL `16.14` を選択してインストールします。

```text
https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
```

インストール時の主な設定は以下を推奨します。

| 項目 | 設定 |
| --- | --- |
| Version | 16.14 |
| Port | 5432 |
| Locale | Default locale |
| Stack Builder | 必須ではないため未選択で可 |

インストール後、PostgreSQL サービスが起動していることを確認してください。

### 5.5 DB 接続設定

Backend の接続設定は `backend/src/main/resources/application.yml` に定義されています。

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:postgresql://localhost:5432/postgres?currentSchema=management_app}
    username: ${DB_USERNAME:postgres}
    password: ${DB_PASSWORD:postgres}
    driver-class-name: org.postgresql.Driver
```

必要に応じて、コマンドプロンプトで以下の環境変数を設定してから起動してください。

```cmd
set DB_URL=jdbc:postgresql://localhost:5432/postgres?currentSchema=management_app
set DB_USERNAME=postgres
set DB_PASSWORD=実際のPostgreSQLパスワード
```

### 5.6 PostgreSQL のスキーマ作成

この手順では、DB 名、ユーザー名、パスワードはすべて `postgres` を前提とします。

PostgreSQL を起動した状態で、管理ユーザー `postgres` から以下を実行します。

```sql
CREATE SCHEMA IF NOT EXISTS management_app AUTHORIZATION postgres;
```

接続確認を行う場合は以下を実行します。

```cmd
psql -h localhost -p 5432 -U postgres -d postgres
```

接続後、スキーマが作成されていることを確認します。

```sql
\dn management_app
```

### 5.7 Backend 起動

```cmd
cd project-management-app\backend
mvn spring-boot:run
```

起動後、以下にアクセスして確認します。

```text
http://localhost:8080
```

Health API が利用できる場合は以下も確認します。

```text
http://localhost:8080/actuator/health
```

## 6. Frontend 環境構築

```cmd

```


起動後、以下にアクセスして確認します。

## 7. 起動順序

ローカルで動作確認する場合は、以下の順序で起動します。

1. PostgreSQL を起動する
2. Backend を `mvn spring-boot:run` で起動する
3. Frontend を `npm run dev` で起動する
4. ブラウザで `http://localhost:5173` を開く

## 9. よくあるトラブル

### Backend が DB 接続エラーで起動しない

以下を確認してください。

- PostgreSQL が起動していること
- `postgres` データベースへ接続できること
- `management_app` スキーマが作成済みであること
- `postgres` ユーザーで接続できること
- `application.yml` の URL / ユーザー名 / パスワードと一致していること
- 必要なら `DB_URL` / `DB_USERNAME` / `DB_PASSWORD` で上書きしていること
- PostgreSQL のポートが `5432` で待ち受けていること

### VS Code で Java プロジェクトが認識されない

以下を試してください。

- `Extension Pack for Java` が有効になっていることを確認する
- `Maven for Java` が有効になっていることを確認する
- コマンドパレットから `Java: Clean Java Language Server Workspace` を実行する
- VS Code を再起動する

### CORS エラーが出る

Frontend の起動 URL が `http://localhost:5173` であることを確認してください。

Backend の CORS 設定では、以下が許可されています。

```yaml
projectmanagementapp:
  cors:
    allowed-origins:
      - http://localhost:5173
```
