# 07 開発ルール

## 共通

- 実在企業名、社内用語、機密情報をコミットしない
- 仕様変更時は docs を同時更新する
- 画面文言は日本語ベース、識別子は英語ベースで統一する

## ブランチ運用

- `main`: 安定版
- `develop`: 開発統合
- `feature/*`: 機能開発
- `fix/*`: 不具合修正

## コーディング方針

- Frontend は Composition API を採用する
- Backend は controller / service / repository の責務を分離する
- DTO を経由して API 入出力を定義する

## レビュー観点

- 命名が一貫しているか
- 公開リポジトリに不適切な情報が入っていないか
- docs と実装に差分がないか
